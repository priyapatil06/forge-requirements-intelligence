# Portfolio claims checklist

Use this file to keep the public case study aligned with evidence.

| Claim | Proven by repository? | Evidence required before publishing |
|---|---:|---|
| React + FastAPI + Claude architecture | Yes | Repository and deployment |
| Six-field intake taxonomy | Yes | UI and API schema |
| Five generated artifact types | Yes | Validated artifact output |
| Confidence-flag layer | Yes | Schema, prompt, UI |
| SQLite local / PostgreSQL Docker | Yes | Configuration and Docker Compose |
| Jira OAuth and issue creation | Implemented, needs your Jira test | Successful sandbox run and screenshots |
| 14-item regression corpus | Yes | `backend/test_corpus/features.json` |
| 127 prompt variations | No | Git history or experiment log with 127 recorded runs |
| Six weeks of prompt iteration | No | Dated experiment log |
| 18 early adopters | No | Recruitment record and anonymized participant log |
| 94% API-contract accuracy | No | Rating rubric, raw ratings, calculation, reviewer profile |
| 4.2x faster ticket creation | No | Raw timing data and analysis |
| 84% recommendation rate | No | Raw survey responses |
| Four-month build | Not established by code | Repository history over that period |

Until the evidence exists, phrase measured outcomes as **planned validation metrics** rather than completed results.
